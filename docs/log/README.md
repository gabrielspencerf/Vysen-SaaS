# Log de erros, falhas e soluções

Registro passo a passo dos problemas encontrados no projeto e das soluções aplicadas. Serve como referência para quem retoma o trabalho ou encontra o mesmo erro.

**Convenções:**
- Cada entrada tem **data/contexto**, **erro/falha**, **causa** e **solução**.
- Novos itens devem ser adicionados ao **topo** da lista em [REGISTRO.md](./REGISTRO.md) (mais recente primeiro).
- Se um problema se repetir em outro contexto, adicione uma nova entrada referenciando a anterior quando fizer sentido.
